# Context: Documentation

## Purpose

Documentation for the Angular frontend template, including feature notes and change records.

## Boundaries

- Owns human-readable documentation and change documentation.
- Does not own Angular runtime code.

## Important Files

- `angular-template.md`: template feature description.
- `changes/README.md`: change documentation rules.

## Local Commands

- `npm run changes`

## Decisions

- Change documentation is validated in CI.
- Agents must document verification and context updates for meaningful changes.
